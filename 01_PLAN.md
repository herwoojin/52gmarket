# 01. PLAN — 오이(52)지마켓

> 사내에서 한쪽은 사고 한쪽은 버리는 전산소모품·사무용품의 낭비를 없애는,
> 직원 간 나눔·재판매 마켓플레이스. 마스코트는 채소 **오이** 🥒.

---

## 1. 한 줄 정의
같은 회사·같은 사무실 안에서 **버려지는 소모품을 다른 직원이 받아 쓰도록** 연결하는 사내 나눔/재판매 웹앱.

## 2. 문제 (Why)
- A팀은 토너·A4용지·케이블을 **구매**하는데, B팀은 같은 물건을 **폐기**한다.
- 본인이 쓰려고 산 물건이 **사용 불가**가 되어도 처분 경로가 없다.
- 정보 비대칭으로 멀쩡한 자원이 낭비되고, 불필요한 구매 비용이 발생한다.

## 3. 해결 (What)
- 직원이 안 쓰는 물건을 **나눔(무료)** 또는 **재판매(단가 지정)** 로 올린다.
- 다른 직원이 **대화창**에서 물건을 확인하고 **미리 찜(오이지 담그기)** 하거나 나눔을 받는다.
- 판매는 **토스/스트라이프/페이팔** 중 간편결제로 실제 거래한다.
- 필요한 물건의 **키워드 알림**을 걸어두면, 매물이 올라올 때 바로 알림을 받는다.

## 4. 타깃 사용자
| 구분 | 설명 |
|---|---|
| 올리는 사람 | 안 쓰는 소모품·비품이 생긴 직원 (각 팀·지역사무소) |
| 받는 사람 | 해당 물건이 필요해 구매 예정인 직원 |
| 관리자 | 매물 데이터를 직접 점검·정리하는 운영 담당(=프로젝트 오너) |

## 5. 핵심 컨셉 — 오이 🥒
- 브랜드 마스코트는 채소 **오이**, 다크 그린 UI.
- **찜하기 = "오이지 담그기"**: 마음에 든 매물을 *항아리*에 절여 모아두는 메타포.
- 친근하고 가벼운 톤으로 사내 참여를 유도.

## 6. 범위 (Scope)

### Phase 1 — MVP (P0)
- 구글 로그인 (Firebase Auth)
- 매물 등록/목록/상세 (구글시트 + Apps Script)
- 사진 업로드 → **WEBP 자동 변환** → Firebase Storage
- 닉네임 + 근무지(픽업 위치) 노출, **실명 비공개**
- 찜(오이지 항아리)
- 모바일/태블릿 반응형, 다크 UI, **PWA**

### Phase 2 — 거래·소통 (P1)
- 매물별 **1:1 대화창** (실시간)
- 키워드 **알림** (매물 업로드 시 매칭 → 인앱/푸시)
- **결제** (토스페이먼츠 우선, 스트라이프/페이팔 옵션)
- 거래완료 처리

### Phase 3 — 운영·확장 (P2)
- 관리자용 시트 사이드바(등록/편집/사진) 고도화
- 지역사무소별 필터·통계(절감액 추정)
- FCM 웹푸시 정식화

### 비범위 (Out of Scope, 초기)
- 외부(사외) 거래, 택배 배송, 정산/세금계산서
- 복잡한 평점·리뷰 시스템

## 7. 성공 지표 (Success Metrics)
- 월간 등록 매물 수, 나눔/판매 성사 건수
- "버려질 뻔한" 물건의 재사용 건수(자기보고)
- 추정 절감액(판매 단가·나눔 정가 합)
- 활성 사용자 비율(올린 사람 / 전체 로그인)

## 8. 리스크 & 대응
| 리스크 | 대응 |
|---|---|
| 결제 토큰이 프론트 노출 | Apps Script에서 Firebase ID 토큰 검증(2차 강화), Storage 규칙 |
| 구글시트 기반의 실시간성 한계 | 채팅/알림은 Firestore 사용, 매물만 시트 |
| 사내 개인정보 노출 우려 | 실명 미노출, 닉네임+근무지만, 본인 매물만 수정 |
| 참여 저조 | 키워드 알림·오이 브랜딩으로 재방문 유도 |
