# 🥒 오이(52)지마켓 — 백엔드 연동 가이드

데이터 단일 출처는 **구글 시트**입니다. 시트를 직접 고치면 앱에 그대로 반영됩니다.

```
[앱]  ──로그인·WEBP 사진──▶  [Firebase Auth / Storage]
[앱]  ──목록·등록(JSON)──▶   [Apps Script Web App]  ◀──읽기·쓰기──▶  [구글 시트 '매물']
                                                                       ▲
                                              [관리자(나)] ──직접편집·사이드바──┘
```

---

## 1. 구글 시트 만들기 — 매물 DB

새 구글 시트를 만들고, 시트 탭 이름을 **`매물`** 로 합니다. (Apps Script가 비어있으면 헤더를 자동 생성하므로 비워둬도 됩니다.)

컬럼 구조(헤더):

| 컬럼 | 의미 | 예시 |
|---|---|---|
| `id` | 고유 ID (자동) | p1719... |
| `createdAt` | 등록 시각 (자동) | 2026-06-30 |
| `status` | 상태 | 판매중 / 거래완료 / 삭제 |
| `deal` | 거래방식 | 나눔 / 판매 |
| `category` | 카테고리 | 전산소모품 / 사무용품 / 가구·비품 / 기타 |
| `title` | 제목 | HP 26A 토너 미개봉 |
| `price` | 단가(원) | 38000 (나눔은 0) |
| `desc` | 설명 | 5층에서 픽업 가능 |
| `loc` | 근무지(픽업 위치) | 본사 3층 IT운영팀 |
| `nick` | 닉네임 | 오이박사 |
| `uid` | 작성자 구글 UID | (로그인 사용자) |
| `photoURL` | 사진 주소 | https://firebasestorage... |
| `jjim` | 찜 수 | 7 |
| `chats` | 대화 수 | 3 |

> 💡 **직접 편집/삭제**: 행 값을 바꾸면 즉시 반영됩니다. 매물을 숨기려면 `status`를 `삭제`로 바꾸거나, 행 자체를 지워도 됩니다.

## 2. Apps Script 붙이기

1. 시트 메뉴 → **확장 프로그램 → Apps Script**
2. `Code.gs` 내용을 붙여넣고, **파일 추가 → HTML**로 `Sidebar` 파일을 만들어 `Sidebar.html` 내용을 넣습니다. (파일명은 확장자 없이 `Sidebar`)
3. **프로젝트 설정 → 스크립트 속성**에 다음을 등록:

| 속성 | 값 |
|---|---|
| `API_TOKEN` | 앱과 공유할 비밀 문자열 (길게 아무거나) |
| `FB_BUCKET` | `프로젝트id.appspot.com` |
| `FB_CLIENT_EMAIL` | 서비스 계정 이메일 (시트 안 사진등록용) |
| `FB_PRIVATE_KEY` | 서비스 계정 private_key 통째로 |

4. **배포 → 새 배포 → 유형: 웹 앱**
   - 실행: **나(소유자)**
   - 액세스 권한: **모든 사용자**
   - 배포하면 나오는 `https://script.google.com/macros/s/...../exec` 주소를 복사 → 앱 설정의 `appsScriptUrl`에 넣습니다.

> 이후 메뉴에 **🥒 오이지마켓**이 생깁니다. 시트 안에서 등록·편집·사진등록·거래완료 처리가 가능합니다.

## 3. Firebase 설정 — 구글 로그인 + 사진 저장

1. [Firebase 콘솔](https://console.firebase.google.com)에서 프로젝트 생성
2. **Authentication → 로그인 방법 → Google 사용 설정**
3. **Storage → 시작하기** (버킷 생성)
4. Storage 규칙 — 로그인 사용자만 업로드:
   ```
   rules_version = '2';
   service firebase.storage {
     match /b/{bucket}/o {
       match /listings/{file} {
         allow read: if true;
         allow write: if request.auth != null;
       }
     }
   }
   ```
5. **프로젝트 설정 → 일반 → 내 앱(웹)** 추가 → `firebaseConfig` 값 복사
6. **승인된 도메인**(Authentication → 설정)에 앱 배포 도메인 추가 (예: netlify 주소)

### 서비스 계정 (시트 안에서 사진 올릴 때만)
**프로젝트 설정 → 서비스 계정 → 새 비공개 키 생성** → 받은 JSON에서 `client_email`, `private_key`를 위 스크립트 속성에 넣습니다.

> 앱에서 올리는 사진은 Firebase JS SDK가 직접 처리하므로 서비스 계정이 필요 없습니다. 서비스 계정은 **관리자가 시트 사이드바에서 직접 사진을 올릴 때만** 사용됩니다.

## 4. 앱에 설정값 넣기

`oiji-market.html` 상단 `OIJI_CONFIG`를 채우고 `configured: true`로 바꾸면 실제 백엔드와 연결됩니다. 비워두면 데모 데이터로 동작합니다.

```js
const OIJI_CONFIG = {
  configured: true,
  appsScriptUrl: "https://script.google.com/macros/s/...../exec",
  apiToken: "여기에 API_TOKEN과 동일한 값",
  firebase: {
    apiKey: "...", authDomain: "...", projectId: "...",
    storageBucket: "프로젝트id.appspot.com", appId: "..."
  }
};
```

## 5. 동작 흐름 정리

**매물 올리기 (앱)**
1. 구글 로그인 → 닉네임·근무지만 노출 (실명 비공개)
2. 사진 선택 → 브라우저에서 WEBP 변환 → Firebase Storage 업로드 → 다운로드 URL 획득
3. 제목·단가 등과 함께 Apps Script `create` 호출 → 시트에 새 행 추가

**매물 보기 (앱)**
- 시작/새로고침 시 Apps Script `doGet` 호출 → 시트의 모든 매물을 불러옴
- `status='삭제'` 행은 자동 제외

**관리 (나)**
- 시트를 직접 수정/삭제 → 앱 새로고침 시 반영
- 또는 **🥒 오이지마켓 메뉴 → 등록·편집 패널**에서 사진 포함 등록/수정

## 보안 메모
- `apiToken`은 프론트엔드에 노출됩니다. 쓰기 남용을 진짜로 막으려면 Apps Script에서 `body.uid`를 Firebase ID 토큰으로 검증하는 단계를 추가하세요(2차 강화).
- 서비스 계정 `private_key`는 **절대 프론트엔드에 넣지 말고** 스크립트 속성에만 둡니다.
- Storage 규칙으로 미인증 업로드를 차단하세요.
