# 02. PRD — 오이(52)지마켓

> 제품 요구사항. 우선순위: **P0**(MVP 필수) / **P1**(2차) / **P2**(추후)

---

## 1. 페르소나

### P1. 올리는 직원 "정리하는 미나"
- 프린터 교체로 미개봉 토너가 남음. 버리긴 아깝고 처분 경로가 없음.
- 사진 찍어 빠르게 올리고, 픽업만 약속되면 끝나길 원함.

### P2. 받는 직원 "구매하려던 준호"
- A4용지를 주문하려다 사내에 나눔이 있는지 먼저 보고 싶음.
- 키워드 알림으로 "토너", "A4용지"가 뜨면 바로 받고 싶음.

### P3. 운영자 "오너(나)"
- 매물 데이터를 **구글시트에서 직접** 보고 정리/삭제하고 싶음.
- 가끔 대표 매물을 직접 등록(사진 포함)하고 싶음.

---

## 2. 사용자 스토리 (핵심)
- (P0) 직원으로서, 구글 계정으로 로그인해 **실명 노출 없이** 닉네임으로 활동하고 싶다.
- (P0) 안 쓰는 물건을 사진과 함께 올리되, 사진이 **자동으로 가벼워지길** 원한다.
- (P0) 매물을 나눔/판매로 구분하고, 판매는 **단가**를 지정하고 싶다.
- (P0) 마음에 든 매물을 **항아리에 담아(찜)** 나중에 다시 보고 싶다.
- (P1) 판매자와 **대화창**에서 상태를 묻고 픽업을 약속하고 싶다.
- (P1) 필요한 물건 **키워드를 등록**해, 올라오면 알림받고 싶다.
- (P1) 판매 물건을 **간편결제**로 바로 결제하고 싶다.
- (P0) **휴대폰에 앱처럼 설치(PWA)** 하고, 모바일·태블릿에서 편하게 쓰고 싶다.

---

## 3. 기능 요구사항

### 3.1 인증/프로필 (P0)
- 구글 로그인/로그아웃 (Firebase Auth).
- 최초 로그인 시 닉네임 입력(기본값: 구글 표시이름 제안), 근무지 선택.
- 프로필: 닉네임·근무지 수정. **실명·이메일은 타인에게 미노출.**
- 본인 매물만 수정/삭제 가능(작성자 `uid` 검증).

### 3.2 매물 등록 (P0)
- 입력: 제목, 카테고리(전산소모품/사무용품/가구·비품/기타), 거래방식(나눔/판매), 단가(판매시), 근무지(픽업), 설명, 사진(선택).
- 사진: 클라이언트에서 **WEBP 변환 + 최대 1280px 리사이즈** 후 업로드. 원본/변환 용량·절감률 표시.
- 등록 시 작성자 닉네임·근무지·uid 함께 기록.
- 유효성: 제목 필수, 판매시 단가>0.

### 3.3 매물 목록/상세 (P0)
- 목록: 카드 그리드(모바일 2열/태블릿 3열), 최신순.
- 필터: 카테고리 칩, 거래방식(전체/나눔/판매) 칩.
- 카드: 썸네일, 제목, 가격/나눔배지, 근무지, 닉네임, 찜수·대화수, 거래완료 배지.
- 상세: 큰 이미지, 설명, 메타(픽업·닉네임·카테고리), 액션(찜/대화/결제).

### 3.4 찜 = 오이지 항아리 (P0)
- 카드/상세에서 찜 토글. 찜한 매물은 **항아리** 탭에 모임.
- 찜 수 카운트 반영. (개인 데이터)

### 3.5 대화창 (P1)
- 매물별 1:1 대화(구매자 ↔ 판매자), 실시간.
- 빠른 답장 칩("아직 가능할까요?", "미리 찜할게요", "픽업 시간은?").
- 대화 내 미리 찜하기.
- 대화 수 카운트 반영.

### 3.6 알림 (P1)
- 키워드 등록/삭제(예: 토너, A4용지).
- 새 매물 제목·설명이 키워드와 매칭되면 **인앱 알림** 생성, 탭 배지 갱신.
- (P2) FCM 웹푸시로 백그라운드 알림.

### 3.7 결제 (P1)
- 판매 매물 결제: **토스페이먼츠**(국내 우선). 옵션으로 스트라이프/페이팔.
- 결제 요약(상품가 + 안전수수료), 수단 선택, 결제 처리 → 거래완료 처리.
- 결제 성공 시 매물 `status=거래완료`, 채팅으로 픽업 안내.

### 3.8 PWA / 반응형 (P0)
- `manifest` + 서비스워커, "홈 화면에 추가" 설치.
- 모바일·태블릿 반응형, 다크 UI, 하단 탭 네비.
- 오프라인 시 캐시된 셸 표시(목록은 온라인 필요).

## 4. 비기능 요구사항
- **성능**: 목록 첫 로딩 2.5s 이내(시트 캐시), 이미지 lazy-load.
- **보안/프라이버시**: 실명 미노출, 본인 데이터만 쓰기, Storage 규칙으로 미인증 업로드 차단, API 토큰 + (P1)ID 토큰 검증.
- **접근성**: 명도 대비 준수, 키보드 포커스, `prefers-reduced-motion` 존중.
- **국문 우선**: 모든 사용자 노출 문구 한국어.
- **호환성**: 최신 모바일 Safari/Chrome, 태블릿.

## 5. 화면 목록 (정보구조)
1. 홈(매물 목록 + 필터)
2. 올리기(등록 폼)
3. 오이지 항아리(찜)
4. 알림(키워드 + 받은 알림)
5. 내정보(로그인/닉네임/근무지)
6. 매물 상세(시트형 모달)
7. 대화창(시트형 모달)
8. 결제(시트형 모달)

## 6. 수용 기준 예시 (Given-When-Then)
- **등록**: *Given* 로그인 상태, *When* 제목+사진으로 등록, *Then* 사진이 WEBP로 Storage에 저장되고 시트에 새 행이 생기며 목록 최상단에 노출.
- **프라이버시**: *Given* 타 직원이 매물을 봄, *When* 상세를 열면, *Then* 닉네임·근무지만 보이고 실명·이메일은 어디에도 없음.
- **알림**: *Given* 키워드 "토너" 등록, *When* 제목에 "토너" 포함 매물 등록, *Then* 알림 목록에 항목 추가 + 탭 배지 +1.

자세한 기술 정의는 `03_TRD`, 데이터 모델은 `04_ERD` 참조.
